      Settings — GitHub token (required to start/update checklists)

      Reads work without a token since this repo is public. To create or update a checklist, each tech needs a GitHub fine-grained personal access token scoped only to this repository with Contents: Read and write permission. Create one at github.com/settings/personal-access-tokens/new, then paste it below. It is stored only in this browser's local storage, never sent anywhere except to GitHub's API.

      Note: this repo is public, so ticket numbers, tech names and timestamps saved here are publicly visible on GitHub. Each checklist action (starting a checklist, marking an item complete) is committed to the repo's history. If that's not acceptable for your data, switch the repo to private in GitHub settings — the app works the same way, but reads will then also require a token.
